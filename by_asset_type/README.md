# by_asset_type/ — One folder per asset type

If you're doing a print batch or auditing one asset type at a time, this is the folder you want. Every diet card lives in `diet_cards/`, every door totem in `door_totems/`, and so on.

## Contents

| Folder | What's inside | Count |
|---|---|---|
| **[diet_cards/](diet_cards/)** | All canonical diet cards (one per species) | 22 |
| **[door_totems/](door_totems/)** | Encounter entry door signs (per site: HOU / SA) | 20+ |
| **[walkby_totems/](walkby_totems/)** | Corridor / aisle pull-tab totems | 24+ |
| **[micro_totems/](micro_totems/)** | Small handheld cards | 23+ |
| **[styled_totems/](styled_totems/)** | Full-glamour styled totems | 22+ |
| **[student_totems/](student_totems/)** | STEM / classroom-format totems | 4 (more pending) |
| **[narration_suite_totems/](narration_suite_totems/)** | 20 narration-suite totems with $300/night rental info | 20 |

## When to use this vs. by_species/

- **Use `by_asset_type/`** when you're printing a batch ("print all diet cards for the new sign run") or auditing one asset type ("are all 22 species' door totems present?").
- **Use `by_species/`** when you're working on one animal ("everything for the leopard tortoise") or onboarding a new staff member to a single exhibit.
