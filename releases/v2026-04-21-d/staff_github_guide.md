# Staff Guide — How to Get the Current LIFE Totem Files

**Purpose:** Any staff member who needs the current printable totems, signs, or documents should get them from GitHub — not from email, not from a thumb drive, not from a personal phone. GitHub is the single source of truth.

**Repo name:** `ammoncovino/LIFE-system` (private)

---

## The 60-second version

1. Open a web browser. Go to **https://github.com/ammoncovino/LIFE-system**
2. Sign in with the GitHub account the owner gave you
3. Click the green **"Code"** button → **"Download ZIP"**
4. Unzip the file on your computer
5. Open `releases/v2026-04-21-d/` for the current totems

That's it. You now have every current file.

---

## Which folder do I want?

When you open the repo, look for `releases/`. Inside that, the newest dated folder is the current one. Today's current release is:

**`releases/v2026-04-21-d/`**

Inside it you'll find:

| If you need... | Open this |
| --- | --- |
| All 19 totems in one PDF to print | `LIFE_Totems_Master_Binder_v2026-04-21-d.pdf` |
| A single animal's totem | `totems/TOTEM_{animal-name}.pdf` |
| The rules and fabrication notes | `README.md` |
| The content files (only edit if you're the owner) | `source/` and `content/` |

---

## Rules for staff

1. **Always use the newest dated folder under `releases/`.** Older folders are kept for history only — do not print from them.
2. **Never edit a PDF.** If something in a totem is wrong, tell the owner. Content gets fixed in the source files and the totems get re-rendered.
3. **Never print from an email attachment or a thumb drive.** If it's not from the current `releases/` folder on GitHub, it's not the current version.
4. **The Homo sapiens totem has a mirror cutout.** Do not print and hang it — it goes to the fabricator with the `README.md` fabrication note attached.

## Signing in to GitHub (first time)

If the owner has not yet given you a GitHub account:

1. Go to **https://github.com/signup**
2. Create an account using your work email
3. Send your GitHub username to the owner
4. The owner will grant you read access to the private `LIFE-system` repo
5. Once you have the invite email, click **"Accept invitation"**

After that, you can sign in at **https://github.com/login** and download any current release.

---

## Updating the files you already downloaded

GitHub keeps updating. When the owner tells you a new release is out:

- Go back to **https://github.com/ammoncovino/LIFE-system**
- Download ZIP again
- Replace your old folder with the new one
- Print from the new folder only

Do not try to merge old files with new ones. The new release is always the complete set.

---

## Troubleshooting

**"I can't see the repo — it says 404"**
The owner hasn't granted your account access yet. Send your GitHub username to the owner.

**"The ZIP is huge / my computer is slow"**
You can download a single release folder only by clicking into `releases/v2026-04-21-d/`, then right-clicking any individual file and choosing "Download." Use this for one-off prints.

**"Which version is current?"**
The newest dated folder in `releases/`. The date format is `v2026-04-21-d` = April 21, 2026 revision d. Letters go up (a, b, c, d) as we ship fixes; dates change when we ship a new set.

**"Can I print v2026-04-21-c instead?"**
No. Older versions are kept for history only. Always print from the newest release.
